# Task Service

Сервис для управления задачами с HTTP API на Go.

## Реализованная фича — Периодические задачи

### Описание
Добавлена возможность создавать периодические задачи для персонала.
Врач или менеджер может создать задачу один раз и указать расписание —
система будет автоматически создавать задачи по заданному расписанию.

### Типы периодичности

| Тип | Описание | Пример |
|-----|----------|--------|
| `daily` | Каждые N дней | Каждый день, каждые 2 дня |
| `monthly` | В определённые числа месяца | 1-го и 15-го каждого месяца |
| `specific_dates` | Конкретные даты | 1 июня, 8 марта |
| `parity` | Чётные или нечётные дни | Только по чётным числам |

---

## Архитектурные решения и допущения

### Подход к генерации задач
Выбран подход с **фоновым заданием (scheduler)** вместо создания
всех задач заранее на весь период. 

**Почему так:**
- Сотрудник может уволиться — достаточно деактивировать настройку
  и новые задачи перестанут создаваться
- БД не захламляется задачами на месяцы и годы вперёд
- Легко приостановить и возобновить периодичность

### Хранение данных
Настройки периодичности хранятся в отдельной таблице `task_recurrences`.
Каждая созданная задача ссылается на настройку через `recurrence_id`.
task_recurrences    tasks
─────────────────   ──────────────────────────
id                  id
title          ←──  recurrence_id
description         title
type                scheduled_date
interval            status
month_days          created_at
dates               updated_at
parity
is_active

### Обратная совместимость
Поле `recurrence` в запросе опциональное — существующий API создания
задач работает без изменений.

## API

### Базовый префикс
/api/v1

### Создать обычную задачу (без изменений)
POST /api/v1/tasks
```json
{
  "title": "Подготовить отчёт",
  "description": "Квартальный отчёт",
  "status": "new"
}
```

### Создать периодическую задачу

**Каждый день:**
```json
{
  "title": "Обзвон пациентов",
  "description": "Ежедневный обзвон",
  "recurrence": {
    "type": "daily",
    "interval": 1
  }
}
```

**Каждые 2 дня:**
```json
{
  "title": "Проверка оборудования",
  "recurrence": {
    "type": "daily",
    "interval": 2
  }
}
```

**По чётным дням:**
```json
{
  "title": "Формирование отчётности",
  "recurrence": {
    "type": "parity",
    "parity": "even"
  }
}
```

**1-го и 15-го числа каждого месяца:**
```json
{
  "title": "Инвентаризация",
  "recurrence": {
    "type": "monthly",
    "month_days": [1, 15]
  }
}
```

**На конкретные даты:**
```json
{
  "title": "Праздничное дежурство",
  "recurrence": {
    "type": "specific_dates",
    "dates": ["2026-01-01", "2026-03-08", "2026-05-09"]
  }
}
```

### Деактивировать периодичность
POST /api/v1/recurrences/{id}/deactivate
Останавливает создание новых задач. Уже созданные задачи остаются.

## Запуск

### Требования
- Docker и Docker Compose

### Быстрый старт
```bash
docker compose up --build
```

После запуска:
- API: `http://localhost:8080`
- Swagger UI: `http://localhost:8080/swagger/`

### Пересоздать БД
```bash
docker compose down -v
docker compose up --build
```
